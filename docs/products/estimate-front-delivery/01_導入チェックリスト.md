# 01_導入チェックリスト

## 1. 最初に確認すること

- [ ] 事業者名、業種、対応エリアを決める
- [ ] 問い合わせ受付メールを決める
- [ ] 公式LINE URLを使うか決める
- [ ] 現地調査の有無を決める
- [ ] 予約金/着手金/現地調査費を案内するか決める
- [ ] 対応できない工事やエリアを明文化する
- [ ] キャンセル・日程変更時の注意事項を決める

## 2. テスト公開前の確認

- [ ] 受付フォームの必須項目が多すぎないか確認する
- [ ] 写真提出の案内が分かりやすいか確認する
- [ ] 自動返信に営業時間と返信目安が入っているか確認する
- [ ] 予約金/着手金を案内する場合、決済前の注意事項が入っているか確認する
- [ ] Square決済リンクを貼る場合、テスト用・本番用を取り違えていないか確認する
- [ ] 顧客向けメールが `TEST_MODE` または下書き/ログ確認で検証済みか確認する

## 3. 公開後の毎日運用

- [ ] 新規受付が台帳に記録されているか確認する
- [ ] 返信が必要な問い合わせを確認する
- [ ] 予約金/着手金を案内する前に、対象工事・対応エリア・日程条件を確認する
- [ ] キャンセルや変更が発生した場合、台帳に履歴を残す

## 4. トラブル予防

- [ ] 売上保証、成約保証、法律判断のような表現を使わない
- [ ] 工事可否や見積金額は自動返信で断定しない
- [ ] 返金、クレーム、法的トラブルは自動返信だけで処理しない
