# BRIDGE 見積前受付フロント MVP仕様書

## 1. 商品名

BRIDGE 見積前受付フロント

## 2. 商品コンセプト

外構・リフォーム・建設系の小規模事業者が、問い合わせ直後に行う「見積前確認」「注意事項案内」「予約金/着手金の案内」「顧客台帳化」を、Googleフォーム相当の受付フォーム、Googleスプレッドシート、Apps Scriptで安全に半自動化する販売検証用MVPです。

本MVPは、既存のProofPack本番フロー、Square本番リンク、Cloudflare Worker、既存GAS本番コードを変更せず、新商品用の独立したLP・仕様書・GAS雛形・テスト手順を作ることを目的とします。

## 3. 誰向けか

- 外構業者
- リフォーム業者
- 建設系一人親方
- 小規模工務店
- 現地調査や見積対応に時間を取られている事業者
- LINE・電話・DMで問い合わせ対応している事業者
- 見積だけで終わる案件に課題を感じている事業者
- 予約金/着手金を自然な流れで案内したい事業者

## 4. 何を自動化するか

- 問い合わせフォーム送信内容の受信
- 見積前に必要な情報の整理
- Leadsシートへの問い合わせ記録
- Mail_Logへの自動返信予定の記録
- Mail_Logへの事業者通知予定の記録
- Settingsシートに登録済みのSquare決済リンク差し込み
- 予約金/着手金が必要な場合のみ案内文へ反映
- Reminder_Logへの未返信リマインド予定の記録
- System_Log / System_DLQへのエラー記録

## 5. 何を自動化しないか

- Square決済リンクの作成・変更・削除
- 本番決済の実行
- 本番メール送信の即時有効化
- 工事可否、見積金額、契約条件の個別判断
- 税務・法律・契約に関する判断
- クレーム対応や交渉の代行
- 既存HPの強制差し替え
- 既存ProofPack本番フローの変更
- 個別制作・個別相談を前提にした運用

## 6. 顧客の悩み

- 問い合わせ対応が遅れて機会損失につながる
- 毎回同じ質問に答えている
- 見積前に必要情報が揃わない
- 現地調査だけして成約しないことがある
- 予約金/着手金を言い出しにくい
- 注意事項やキャンセル条件を伝え忘れる
- 顧客情報がLINE・電話・メモに散らばる
- 口約束や認識違いが起きやすい

## 7. 競合との違い

| 比較対象 | 一般的な特徴 | 本商品の違い |
| --- | --- | --- |
| 汎用フォーム作成ツール | 入力受付はできるが、建設系の見積前導線は別途設計が必要 | 外構・リフォーム・建設系の見積前確認に絞ったテンプレートを用意 |
| CRM | 機能が多く、初期設定や運用が重くなりやすい | 小規模事業者が最低限の項目で始められる |
| 予約システム | 日程調整中心で、注意事項や予約金案内は弱い | 見積前確認、注意事項、予約金/着手金案内を一連の受付導線にする |
| 個別制作 | 柔軟だが費用と納期が重くなりやすい | 販売検証用のMVPとして、共通テンプレートで低リスクに始める |

## 8. MVP機能

1. 販売検証用LP
   - 独立ページとして `products/estimate-front/index.html` に配置
   - 申し込みボタンは仮URL `#apply`
   - 本番Squareリンクは入れない
2. 初期設定シート
   - 購入者が入力する項目を選択式中心に整理
3. GAS雛形
   - `TEST_MODE=true` 前提
   - `doPost(e)` で問い合わせを受信
   - Leadsに保存
   - Mail_Logに顧客向け・事業者向け文面を記録
   - Reminder_Logに予定を記録
   - System_Log / System_DLQにエラーを記録
4. README_DEPLOY
   - 非エンジニアでも次の作業が分かる手順書

## 9. 初期設定項目

必須項目はできるだけ少なくし、自由入力を減らします。

- 事業者名
- 業種
- 対応エリア
- 問い合わせ受付メール
- 公式LINE URL
- Square決済リンク
- 予約金または着手金の金額
- 現地調査の有無
- 対応できない工事
- 営業時間
- 返信目安
- キャンセル注意事項

## 10. 料金案

販売検証では、以下の3案をLPや商談でテストします。

| プラン | 価格案 | 内容 |
| --- | ---: | --- |
| MVP導入キット | 29,800円（税込） | 初期設定シート、受付文面、GAS雛形、手順書 |
| 月額ライト | 月額4,980円（税込） | 受付導線のテンプレ利用と軽微な更新想定 |
| 初期設定代行オプション | 19,800円（税込） | 初期入力の代行。個別コンサルではなく設定作業に限定 |

価格は販売検証用の案であり、本番販売前に原価・サポート範囲・責任範囲を再確認します。

## 11. 禁止表現

販売ページ、SNS、メール、LINE、説明資料では以下を使いません。

- 売上保証
- 成約保証
- 必ず入金
- 未払いを防げる
- 法的に守れる
- クレームゼロ
- 完全放置で儲かる
- 税務・法律・契約判断
- 個別コンサル込み

## 12. 受け入れ条件

- 既存本番フローを変更していない
- 新商品用ファイルが分離されている
- LPが独立ページとして作成されている
- 本番Squareリンクが入っていない
- GASがTEST_MODE前提になっている
- 本番メールが誤送信されない設計になっている
- 問い合わせデータをLeadsに記録できる
- Mail_Logに自動返信予定を残せる
- エラーをSystem_LogまたはDLQに残せる
- 非エンジニアでもREADMEを読めば次の作業が分かる

## 14. 完全自動化のための制約

完全自動化には以下の制約があります。

- 工事内容や現場状況は個別性が高く、最終判断は事業者が行う必要がある
- Square決済リンクは事業者側で作成し、Settingsへ貼り付ける必要がある
- メール送信を本番化する前に、送信先、文面、免責、キャンセル条件の確認が必要
- 法律・税務・契約の判断は本商品の対象外
- 顧客の入力ミスや虚偽情報までは防げない
- Google Workspace / Gmailの送信上限に依存する

## 15. リスクと対策

| リスク | 対策 |
| --- | --- |
| 本番メールの誤送信 | 初期値をTEST_MODE=trueにし、テスト時はMail_Log記録のみ |
| 本番Squareリンクの誤利用 | LPは `#apply` のみ。GASはSettingsの値を読むだけで作成・変更しない |
| 既存システムへの影響 | 新規ディレクトリ `products/estimate-front` と `apps-script/estimate-front` に分離 |
| 過度な期待を招く表現 | 禁止表現リストを仕様書・販売文面・READMEで明記 |
| 顧客情報の管理漏れ | Sheetsの台帳化とログ記録を前提にする |
| GAS実行エラー | System_LogとSystem_DLQに記録し、後から手動確認できるようにする |
